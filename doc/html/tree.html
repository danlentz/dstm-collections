rb-tree   inherits from abstract-quad  [Class]

Sets are represented by balanced binary trees The heights of children differ by at most 2. Tree nodes are quadruples (l v r h) where: l = left child, v = value, r = right child, h = height

a    [Slot]

left child of this node containing all elements less than v

b    [Slot]

the value or value-cell represented by this node

c    [Slot]

right child of this node containing all elements greater than v

d    [Slot]

the height of this node, as the distance in child links to the furthest leaf. Rather than annotation using literal :red :black properties, this red-black tree implementation maintains balance based on the relative heights of left and right subtrees, which may never exceed 2.

create   l v r &optional (hl (height l)) (hr (height r))  [Function]

create a tree node with left son l, value v, and right son r.

Must have all elements of l < v < all elements of r. l and r must be balanced and have a height difference =< 2

bal   l v r  [Function]

bal -- similar to create, but also performs one step of rebalancing, if necessary.

assumes l and r balanced and height difference <= 3

join   l v r  [Function]

join -- same as create and bal, but no assumptions are made on the relative heights of l and r

merge   t1 t2  [Function]

merge -- merge two trees l and r into one.

All elements of l must precede the elements of r Assume height difference <= 2

concat   t1 t2  [Function]

concat - merge two trees l and r into one.

All elements of l must precede the elements of r. No assumptions on the heights of l and r.
